# Implementation Plan: Preflight Check

**Branch**: `002-preflight-check` | **Date**: 2026-03-11 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `specs/002-preflight-check/spec.md`

## Summary

Add a preflight readiness validation system to agent-transfer that inventories dependencies on export (manifest.json), validates the target environment on import, and provides a standalone `preflight` CLI command. Reuses existing parsing infrastructure (tool_checker, skill_parser, config_manager, pathfinder) and adds new scanners for CLI tools, env vars, git repos, compiled binaries, and Docker deps.

## Technical Context

**Language/Version**: Python >= 3.8 (matches constitution)
**Primary Dependencies**: Click (CLI), Rich (TUI), PyYAML (frontmatter + .preflight.yml), pathlib, struct, configparser (all stdlib except Rich/Click/PyYAML which are already deps)
**Storage**: JSON files (manifest.json in archives)
**Testing**: pytest
**Target Platform**: Linux + WSL (R3)
**Project Type**: CLI library extension
**Performance Goals**: < 3 seconds for preflight on 50 agents + 20 skills
**Constraints**: No new external dependencies. Reuse existing parsing code.
**Scale/Scope**: Adds ~6 new modules in a `preflight/` subpackage, modifies 2 existing modules (cli.py, transfer.py)

```text
│   ├── preflight/
│   │   ├── __init__.py             # Public API: collect_inventory, run_preflight_checks, etc.

│   │       ├── mcp_scanner.py      # MCP server + provenance detection
│   │       ├── script_scanner.py   # CLI tools + env vars from scripts
│   │       ├── binary_scanner.py   # ELF architecture detection
│   │       ├── git_scanner.py      # Git remote URL extraction
│   │       ├── docker_scanner.py   # Dockerfile/compose detection
│   │       └── preflight_yml.py    # .preflight.yml reader
│   ├── transfer.py                 # Modified: bundle manifest on export
│   └── [existing modules unchanged]

tests/
├── test_preflight.py               # Main test file
├── test_preflight_collector.py     # Collector unit tests
├── test_preflight_checker.py       # Checker unit tests
└── test_preflight_scanners.py      # Individual scanner tests
```

**Structure Decision**: New `preflight/` subpackage under `utils/` with a `scanners/` sub-subpackage. Each scanner is an independent module that can be tested in isolation. The collector composes scanners. The checker validates against scanner output. This matches R9 (plugin architecture) — new scanners can be added without modifying existing code.
